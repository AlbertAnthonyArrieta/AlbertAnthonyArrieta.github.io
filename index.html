CARS Volunteer Management System

Web Application

Technologies Used: Angular/Bootstrap/TypeScript/Java/SpringBoot/MySQL/Git

Summary

A volunteer management system created for the non-profit organization Calgary Animal Rescue Society (CARS). The purpose of the system is to allow CARS administrators to manage volunteer information, foster information, volunteer applications, foster applications, and manage events that active volunteers can register for. This project originally started as a capstone project for SAIT and now has become a volunteer project that I and a few of my SAIT project members have decided to continue to work for CARS after graduation. This project is still currently being worked on and more information can be found in the links below.

My Role

My role during this project was to work with the front-end along with 3 other group members. During the preparation stage, I lead in designing the UI and the prototype interface design to base our project on. During development I had to learn Angular and Bootstrap on my own with some help using the internet and my peers.
Most of my contributions during development was to:

  • Develop CRUD services that communicated with the SQL backend which uses a SpringBoot RESTful API.
  • Creating and deciding how information is laid out throughout each page and coming up with design solutions to ensure an easy navigation experience for the client.
  • Implementation and design of a responsive interface for different screen sizes.
  • Suggesting features that would minimize work needed and increase efficiency for the clients managerial needs.

What I've learned

This project was the first project where I was able to work with a team of developers. This project also consisted of learning new technologies that was never taught before in SAIT such as Angular, TypeScript and Bootstrap. This project forced me to learn independently and taught me that it is an essential skill as a developer to learn new technologies. This introduced me to many new concepts that I have never learnt before in my past programming experience such as component-based development which is a big deal in angular. I also learned a lot of managerial concepts since I was able to work with 6 other students on this project. Iâ€™ve learned how a project development life cycle would look like and how work is distributed to different teams and members within the group. I learnt that communication and planning is critical to the project because with a big group, ideas will get thrown around everywhere and everyone must be on the same page with each other. People will have different opinions on things, and it is important because the group itself will have to come to a consensus on how things will be developed.

Caption-Me Discord Bot

Discord Bot

Technologies Used: JavaScript, Node.js, Heroku

Summary

A small fun project I decided to take on for a discord server that I was in. There would be times where my friends and I in the server would send old and embarrassing photos of ourselves. It was only a matter of time where we had no more new pictures to look at and things would go stale. I sparked up the idea to reuse these pictures that we have found in our computers by simply creating a "random caption" generator. In a span of 2 days I delivered and developed a discord bot using Discord.js API that combines an image and a caption from a list to create something humorous to look at. After I released it into the discord server, my friends and I had a lot of fun with it! And to this day I still add new images and captions to keep the fun going whenever things go stale.
